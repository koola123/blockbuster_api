URL Endpoints of the Blockbuster Movies API:

Business Logic URL HTTP Method Request body data format Response body data format
[1] Return a list of ALL movies to the user /movies GET none A JSON object holding data about ALL movies that were requested.
[2] Return data (description, genre, director, image URL,
whether it’s featured or not) about a single movie by title to the user		 /movies/:title GET none A JSON object holding data about a single movie that was requested.
[3] Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/genres/:title GET none A JSON object holding data about ALL movies that were requested.
[4] Return data about a director (bio, birth year, death year) by name /directors/:name GET none Returns a JSON object.
[5] Allow new users to register /users POST Text message the the user was registered. none
[6] Allow users to update their user info (username) /users/:username PUT none Returns a JSON object about with the new user data.
[7] Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) /users/:username/movies/:movieID POST A JSON object containing movie data. Delivers an updated JSON Object with ALL user movie favorites.
[8] Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later) /users/:username/movies/:movieID DELETE Sends an JSON object. Text message that the desired movie was deleted.
[9] Allow existing users to deregister (showing only a text that a user email has been removed—more on this later) /users/:username DELETE none Text message that the user was deleted.
